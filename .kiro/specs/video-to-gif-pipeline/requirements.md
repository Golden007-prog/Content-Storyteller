# Requirements Document

## Introduction

The Video-to-GIF Pipeline replaces the current unreliable direct GIF generation approach with a two-step architecture: first generate a short loopable video clip, then convert that video to GIF as a post-processing step. This architectural change treats GIF output as a derived artifact from video generation rather than a standalone model output. The pipeline flow becomes: ProcessInput → GenerateCopy (optional) → GenerateLoopVideo (when wantsGif or wantsVideo) → ConvertVideoToGif (when wantsGif) → ComposePackage. The feature preserves all existing Batch Mode behavior and API contracts while adding loop-video planning, a dedicated conversion stage, and updated result schemas to track derived GIF assets alongside their source videos.

## Glossary

- **Pipeline_Runner**: The sequential stage executor at `apps/worker/src/pipeline/pipeline-runner.ts` that runs pipeline stages conditionally based on the resolved OutputIntent.
- **Output_Intent_Planner**: The module at `apps/api/src/services/planner/output-intent.ts` that resolves which pipeline stages to execute based on user preferences, platform defaults, and prompt keywords.
- **GenerateLoopVideo_Stage**: A new or refactored pipeline stage that generates a short (3–8 second) loopable video clip optimized for GIF conversion, using the `videoFast` model slot by default.
- **ConvertVideoToGif_Stage**: A new pipeline stage that takes a generated video asset and converts it to an optimized looping GIF file, handling size limits, temp file cleanup, and timeout protection.
- **Loop_Motion_Planner**: A planning component within the GenerateLoopVideo_Stage that produces a motion brief specifying duration target, looping style, and animation type based on content and platform.
- **GIF_Asset**: A looping animated GIF file derived from a video asset, with metadata including url, mimeType, width, height, durationMs, loop flag, fileSizeBytes, and derivedFromVideoAssetId.
- **Video_Asset**: A short video clip (MP4 or WebM) generated by the GenerateLoopVideo_Stage, serving as the source for GIF conversion.
- **Output_Preference_Selector**: The frontend component at `apps/web/src/components/OutputPreferenceSelector.tsx` that lets users choose their desired output combination in Batch Mode.
- **OutputDashboard**: The frontend component at `apps/web/src/components/OutputDashboard.tsx` that renders generated content results including copy, images, video, and GIF previews.
- **Result_Schema**: The shared type definitions in `packages/shared/src/types/job.ts` that define Job, OutputIntent, AssetReference, and related structures.
- **Model_Router**: The centralized routing layer at `packages/shared/src/ai/model-router.ts` that maps AI capability slots to Vertex AI models.
- **Capability_Registry**: The registry at `apps/worker/src/capabilities/capability-registry.ts` that manages available generation capabilities.

## Requirements

### Requirement 1: Redefine GIF Architecture as Video-First Pipeline

**User Story:** As a user, I want GIF output to be generated by first creating a short video and then converting it to GIF, so that GIF generation is reliable and produces high-quality looping animations.

#### Acceptance Criteria

1. WHEN `wantsGif` is true, THE Pipeline_Runner SHALL set `wantsVideo` to true internally before stage execution begins, ensuring the video generation path is always used as the source for GIF output.
2. THE Pipeline_Runner SHALL execute the GenerateLoopVideo_Stage before the ConvertVideoToGif_Stage in the pipeline execution order.
3. THE ConvertVideoToGif_Stage SHALL accept a Video_Asset produced by the GenerateLoopVideo_Stage as its input and produce a GIF_Asset as its output.
4. THE Pipeline_Runner SHALL NOT execute any direct GIF model generation path; GIF output SHALL always be derived from a video conversion step.
5. WHEN `wantsGif` is true and `wantsVideo` is false in the user-facing OutputIntent, THE Pipeline_Runner SHALL generate the loop video internally but only expose the GIF_Asset in the final output, treating the Video_Asset as an intermediate artifact.
6. WHEN `wantsGif` is true and `wantsVideo` is true, THE Pipeline_Runner SHALL expose both the Video_Asset and the derived GIF_Asset in the final output.

### Requirement 2: Loop-Video Motion Planning

**User Story:** As a user, I want the system to plan a short loopable animation based on my content and target platform, so that the generated GIF looks natural and engaging.

#### Acceptance Criteria

1. WHEN `wantsGif` is true, THE Loop_Motion_Planner SHALL generate a motion brief containing a duration target between 3 and 8 seconds (inclusive).
2. THE Loop_Motion_Planner SHALL select a looping style from the set: "seamless", "bounce", or "restart", defaulting to "seamless" when the content supports smooth looping.
3. THE Loop_Motion_Planner SHALL select an animation type from the set: "workflow_highlight", "subtle_zoom_pan", "text_callout_reveal", "feature_spotlight", "animated_diagram", or "process_flow_loop".
4. WHEN the target platform is LinkedInLaunchPost, THE Loop_Motion_Planner SHALL default to square (1:1) aspect ratio.
5. WHEN the target platform is InstagramReel, THE Loop_Motion_Planner SHALL default to vertical (9:16) aspect ratio.
6. WHEN no specific platform preference is detected, THE Loop_Motion_Planner SHALL default to square (1:1) aspect ratio.
7. THE Loop_Motion_Planner SHALL classify the uploaded content type and select the animation type based on the classification: diagrams and workflows map to "workflow_highlight", UI screenshots map to "feature_spotlight", charts and infographics map to "text_callout_reveal", and unclassified content maps to "subtle_zoom_pan".

### Requirement 3: GenerateLoopVideo Stage

**User Story:** As a user, I want the system to generate a short video clip optimized for looping and GIF conversion, so that the resulting GIF is smooth and compact.

#### Acceptance Criteria

1. THE GenerateLoopVideo_Stage SHALL use the `videoFast` model slot from the Model_Router by default for loop video generation.
2. WHEN the user explicitly requests high-quality output, THE GenerateLoopVideo_Stage SHALL use the `videoFinal` model slot instead of `videoFast`.
3. THE GenerateLoopVideo_Stage SHALL generate a video clip with a duration between 3 and 8 seconds (inclusive) as specified by the motion brief.
4. THE GenerateLoopVideo_Stage SHALL attach video asset metadata containing: url, mimeType, duration, width, height, and a `loopFriendly` flag set to true.
5. THE GenerateLoopVideo_Stage SHALL persist the generated Video_Asset to Cloud Storage under the path `{jobId}/loop-video/{assetId}.mp4`.
6. THE GenerateLoopVideo_Stage SHALL record an AssetReference with assetType `video` in the job asset list upon successful completion.
7. IF the `videoFast` model slot is unavailable, THEN THE GenerateLoopVideo_Stage SHALL record a FallbackNotice and persist the motion brief as a creative direction JSON asset.
8. THE GenerateLoopVideo_Stage SHALL select the aspect ratio based on the motion brief: square (1:1) for LinkedIn and general platforms, vertical (9:16) for Instagram.

### Requirement 4: ConvertVideoToGif Stage

**User Story:** As a user, I want the system to convert the generated video into an optimized GIF file, so that I receive a lightweight looping GIF suitable for social media.

#### Acceptance Criteria

1. THE ConvertVideoToGif_Stage SHALL accept a Video_Asset from the GenerateLoopVideo_Stage and convert it to GIF format.
2. THE ConvertVideoToGif_Stage SHALL optimize the GIF output to a maximum file size of 5 MB (5,242,880 bytes).
3. THE ConvertVideoToGif_Stage SHALL preserve the looping behavior from the source video in the output GIF.
4. THE ConvertVideoToGif_Stage SHALL persist the completed GIF_Asset to Cloud Storage under the path `{jobId}/gifs/{assetId}.gif`.
5. THE ConvertVideoToGif_Stage SHALL record an AssetReference with assetType `gif` and a `derivedFromVideoAssetId` field referencing the source Video_Asset.
6. IF the video-to-GIF conversion fails, THEN THE ConvertVideoToGif_Stage SHALL return the source MP4 or WebM video with a warning, mark the GIF conversion as failed in a FallbackNotice, and continue pipeline execution without blocking.
7. THE ConvertVideoToGif_Stage SHALL enforce a conversion timeout of 60 seconds; IF the timeout is exceeded, THEN THE ConvertVideoToGif_Stage SHALL abort the conversion and apply the failure fallback behavior.
8. THE ConvertVideoToGif_Stage SHALL clean up all temporary files created during conversion, regardless of success or failure.
9. THE ConvertVideoToGif_Stage SHALL log conversion details including source video duration, output GIF file size, conversion duration, and success or failure status.
10. THE ConvertVideoToGif_Stage SHALL enforce a maximum source video duration of 10 seconds; IF the source video exceeds 10 seconds, THEN THE ConvertVideoToGif_Stage SHALL truncate the video to 10 seconds before conversion.

### Requirement 5: Output Intent Rules for GIF Requests

**User Story:** As a user, I want the system to correctly interpret my GIF-related prompts and output preferences, so that the right pipeline stages are triggered without unnecessary processing.

#### Acceptance Criteria

1. WHEN the user selects the "Copy + GIF" output preference, THE Output_Intent_Planner SHALL set `wantsGif` to true, `wantsCopy` to true, and `wantsVideo` to true internally (for the loop video source), while keeping `wantsImage` false.
2. WHEN the user prompt contains any of the keywords "gif", "looping gif", "loop video", "animated explainer", "animated workflow", "short loop", "linkedin gif", "turn this into a gif", or "make this loop" (case-insensitive), THE Output_Intent_Planner SHALL set `wantsGif` to true.
3. WHEN `wantsGif` is set to true by any detection method, THE Output_Intent_Planner SHALL also set `wantsVideo` to true internally to ensure the video generation path is activated.
4. WHEN the user selects the "Full Package" output preference, THE Output_Intent_Planner SHALL set `wantsGif` to true in addition to all other existing intent flags.
5. THE Output_Intent_Planner SHALL NOT generate a standalone image when only "Copy + GIF" is requested.

### Requirement 6: Batch Mode UI Output Options

**User Story:** As a user, I want to select GIF-related output options in the Batch Mode output selector, so that I can explicitly request GIF output alongside copy.

#### Acceptance Criteria

1. THE Output_Preference_Selector SHALL display the following options: "Auto-detect", "Copy only", "Copy + Image", "Copy + GIF", "Copy + Video", and "Full Package".
2. WHEN the user selects "Copy + GIF", THE Output_Preference_Selector SHALL emit the value `copy_gif` as the OutputPreference, triggering `wantsCopy` and `wantsGif` to true and internal `wantsVideo` to true.
3. THE Output_Preference_Selector SHALL render the "Copy + GIF" option with a description of "Text with animated GIF" and a visual style consistent with the existing premium layout.
4. THE Output_Preference_Selector SHALL NOT trigger standalone image generation when "Copy + GIF" is selected.

### Requirement 7: Result Schema for Derived GIF Assets

**User Story:** As a developer, I want the result schema to support GIF assets that are derived from video assets, so that the frontend and API can track the relationship between source videos and derived GIFs.

#### Acceptance Criteria

1. THE Result_Schema SHALL include a `derivedFromVideoAssetId` optional string field on the GIF_Asset metadata type, referencing the source Video_Asset assetId.
2. THE Result_Schema SHALL support `requestedOutputs`, `completedOutputs`, `skippedOutputs`, and `warnings` arrays on the Job type.
3. THE Result_Schema SHALL include a `generateLoopVideo` step and a `convertVideoToGif` step in the StepsMap interface with the same StepMetadata structure as existing steps.
4. THE Result_Schema SHALL include `GeneratingLoopVideo` and `ConvertingVideoToGif` values in the JobState enum.
5. THE Result_Schema SHALL represent each GIF_Asset with the fields: assetId, url, mimeType ("image/gif"), width, height, durationMs, loop (boolean), derivedFromVideoAssetId, and fileSizeBytes.
6. THE Result_Schema SHALL represent each Video_Asset with the fields: assetId, url, mimeType, width, height, duration, and loopFriendly (boolean).

### Requirement 8: Frontend Result Rendering for GIF and Video

**User Story:** As a user, I want to see my generated GIF preview alongside the source video and post copy in the results dashboard, so that I can review and export the complete output.

#### Acceptance Criteria

1. WHEN a GIF_Asset is present in the job results, THE OutputDashboard SHALL render an inline GIF preview that plays the animation in a loop.
2. WHEN both a GIF_Asset and its source Video_Asset are present, THE OutputDashboard SHALL display the source loop video as a secondary collapsible asset below the GIF preview.
3. WHEN the "gif" output type is in the skippedOutputs array, THE OutputDashboard SHALL NOT render a skeleton placeholder for the GIF section.
4. WHEN the GIF conversion failed but the source Video_Asset is available, THE OutputDashboard SHALL display a warning message and render the source video as a fallback.
5. THE ExportPanel SHALL include both the GIF_Asset and the optional source Video_Asset in the downloadable asset list with appropriate labels ("GIF" and "Source Video").
6. THE OutputDashboard SHALL NOT display endless skeleton placeholders for sections that were skipped or failed.

### Requirement 9: Prompt Detection for GIF and Loop Video Requests

**User Story:** As a user, I want the system to detect GIF and loop video intent from natural language prompts, so that I do not need to manually select output preferences for common GIF requests.

#### Acceptance Criteria

1. WHEN the user prompt contains the phrase "gif" (case-insensitive, word boundary), THE Output_Intent_Planner SHALL set `wantsGif` to true.
2. WHEN the user prompt contains any of the phrases "looping gif", "loop video", "animated explainer", "animated workflow", "short loop", "linkedin gif", "turn this into a gif", "make this loop", or "export gif" (case-insensitive), THE Output_Intent_Planner SHALL set `wantsGif` to true.
3. WHEN the user prompt contains "gif" but also contains explicit video keywords ("video", "reel", "teaser", "promo clip"), THE Output_Intent_Planner SHALL set both `wantsGif` and `wantsVideo` to true, producing both a user-visible video and a derived GIF.
4. WHEN the user prompt contains only GIF keywords and no explicit video keywords, THE Output_Intent_Planner SHALL set `wantsGif` to true and keep the user-facing `wantsVideo` false (the internal loop video is still generated but treated as intermediate).

### Requirement 10: Conversion Implementation Constraints

**User Story:** As a developer, I want the video-to-GIF conversion to be reliable across local development and cloud runtime environments, so that the feature works consistently in all deployment contexts.

#### Acceptance Criteria

1. THE ConvertVideoToGif_Stage SHALL support execution in both local development environments and Google Cloud Run production environments.
2. THE ConvertVideoToGif_Stage SHALL enforce a maximum output GIF file size of 5 MB; IF the initial conversion exceeds 5 MB, THEN THE ConvertVideoToGif_Stage SHALL reduce quality or frame rate and retry once before falling back to the source video.
3. THE ConvertVideoToGif_Stage SHALL clean up all temporary files (intermediate frames, palette files, temp video segments) after conversion completes or fails.
4. THE ConvertVideoToGif_Stage SHALL log the following conversion metrics: source video duration in milliseconds, output GIF file size in bytes, conversion wall-clock time in milliseconds, and final success or failure status.
5. THE ConvertVideoToGif_Stage SHALL enforce a maximum source clip duration of 10 seconds for conversion input.

### Requirement 11: Pipeline Stage Ordering and Non-Critical Behavior

**User Story:** As a developer, I want the new pipeline stages to integrate into the existing pipeline runner with correct ordering and non-critical failure handling, so that GIF conversion failures do not break the entire job.

#### Acceptance Criteria

1. THE Pipeline_Runner SHALL execute stages in the order: ProcessInput → GenerateCopy → GenerateImages → GenerateVideo → GenerateLoopVideo → ConvertVideoToGif → ComposePackage.
2. THE Pipeline_Runner SHALL treat the GenerateLoopVideo_Stage as a non-critical stage; a failure SHALL result in a warning and partial completion rather than a full job failure.
3. THE Pipeline_Runner SHALL treat the ConvertVideoToGif_Stage as a non-critical stage; a failure SHALL result in a warning and partial completion rather than a full job failure.
4. WHEN the GenerateLoopVideo_Stage is skipped (because `wantsGif` and `wantsVideo` are both false), THE Pipeline_Runner SHALL record the step status as "skipped" in the StepsMap.
5. WHEN the ConvertVideoToGif_Stage is skipped (because `wantsGif` is false), THE Pipeline_Runner SHALL record the step status as "skipped" in the StepsMap.
6. THE Pipeline_Runner SHALL enforce the existing 10-minute global timeout across all stages including the new GenerateLoopVideo_Stage and ConvertVideoToGif_Stage.
7. THE SSE stream SHALL include the new stage state transitions ("generating_loop_video", "converting_video_to_gif") in state_change events.
